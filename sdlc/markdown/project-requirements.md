# Airline Booking and Management System Project Requirements

This document lists all functional and non-functional requirements of the airline system.

## Functional Requirements

These requirements specify how the airline website, booking, and aircraft management system, henceforth collectively referred to as the system, should be behave.


| Requirement ID | Requirement Description |
|:---------------|:------------------------|
| FR-01 | Customers should be able to search and browse flights without logging in or creating and account. |
| FR-02 | Customers should be able to create an account in order to book a flight. |
| FR-03 | The system should distinguish between user and administrator accounts. |
| FR-04 (Group) | The system must provide an interface to administrators which permits administrators to view and modify privileged information. |
| FR-04-01 | The system's administrator interface must display the status and detailed information of all aircraft in the database. |
| FR-04-02 | The system's administrator interface must display the status and detailed information of all upcoming trips. |
| FR-04-03 | The system's administrator interface must allow administrators to modify the status of an aircraft. |
| FR-04-04 | The system's administrator interface must allow administrators to cancel a trip if it has not started. |
| FR-04-05 | The system must ensure any changes made by the administrator via the administrator interface are reflected in the database. |
| FR-05 | Customers must be able to specify the origin and departure airports when searching for flights. |
| FR-06 | Customers must be able to specify the number of passengers when searching for flights. |
| FR-07 | The website homepage must display promotions pertaining to products, which includes deals on vacations and accommodation. |
| FR-08 | The website homepage must provide a dedicated page which displays the status of all trips, within a specified time window, from the fictional airline. |
